# Work log — 2026-07-15

## Marketing SEO foundation + 6 live Solutions pages ✅ DONE (deployed, submitted to Search Console)

Full write-up: `stemfra_client/docs/SEO_AND_SOLUTIONS.md` (read that first for
architecture). This log is the session summary across all three repos.

**Why:** competitor SEO scan (Mindbody/Squarespace/Lovable/Replit/Wix) found
stemfra.com serving crawlers an 880-byte JS shell with a stale title, while
every competitor serves keyword-rich HTML plus per-vertical "solution"
landing pages. Two-phase fix: (1) real SEO metadata + build-time prerendering,
(2) six `/solutions/<vertical>` pages modeled on Squarespace's pattern.

**`stemfra_client`:**
- SEO metadata single source `data/seo.js` (title/description/JSON-LD per
  route, keyword-first title formula) + `components/Seo.jsx` (runtime head
  updates on route change).
- Build-time prerendering: `entry-server.jsx` (SSR entry via
  `createStaticHandler`/`createStaticRouter`) + `scripts/prerender.mjs`
  (renders all 15 routes → static `dist/<path>/index.html` + `sitemap.xml`;
  `unhideMotion()` fixes framer-motion `opacity:0` initial states so crawlers
  see real text; SSR failures degrade to a head-only shell rather than
  failing the build). `npm run build` is now the 3-step pipeline. `robots.txt`
  added.
- Six Solutions pages (`data/solutions.js` + `pages/Solutions.jsx`), all
  `live:true`: barbers (built first, established the pattern), then
  salons/crossfit/yoga/massage/spa (content + photo-sourcing delegated to 4
  parallel research agents, one per vertical, each independently
  writing+verifying its own hero/problems/features/faqs/closing — cross-
  checked afterward and caught 2 real issues: a duplicate photo reused across
  two verticals, and a vertical's "found on Google" photo showing signage
  branded as a different business type). Page built from lux/Blazeride
  design patterns (`ExperienceSection` hero card, `BusinessSection` problems
  showcase, `EditorialMoment` live-demos band) re-skinned in Stemfra's
  register, plus new shared primitives: `TemplateCard.jsx` (extracted from
  the `/templates` showroom so both surfaces share one card + the desktop
  preview-modal + Start-signup functionality), `CardMarquee.jsx`,
  `JumpPill.jsx` (also now shared by the home page and `/templates`),
  `ServicesMarquee`/`FeatureMarquee.jsx` (live demo-service data).
- Nav reordered (Home·Products·Templates·Solutions | Contact·Pricing·Our
  story·FAQ); home "Designed for your business." section restructured into
  Beauty/Fitness/Wellness category blocks with a jump-pill selector + scroll-
  spy (tiles still link to `/templates/<vertical>`, not Solutions — kept as
  the browse-by-industry entry point per Peter's call); footer wordmark
  matched to the navbar's font/case.
- All 36 Solutions-page images (6 verticals × hero + 5 problems) made
  CRM-managed (`marketing_assets` slots, group `solutions`) with the
  hardcoded URLs kept as code-level fallback — see CRM section below.
- Verified: full `npm run build` run clean, all 15 routes prerendered with
  real titles/content, sitemap complete, all 6 Solutions pages carry valid
  `FAQPage` JSON-LD.
